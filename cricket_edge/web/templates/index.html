How to use Cricket Edge

Cricket Edge is a paper-only research dashboard. It records simulated positions for research; it does not place real-money bets. A blocked gate is useful information, not an error to work around.

Daily workflow

Work in this order

  1. Check Command. Read the Paper Betting Gate and resolve any red blockers first.
  2. Refresh data. Use Fetch Live Odds, then confirm odds are fresh in Data Health.
  3. Run Morning. This updates the paper workflow only when the required data and model checks pass.
  4. Review Opportunities. Inspect fixtures, prices, predicted probability, edge, confidence, and skip reasons.
  5. Monitor and settle. Use Monitor Tick for open paper positions and Settle Paper only after results are available.

Safety rules

What the system will not do

  • It has no real-money betting connector.
  • Demo fixtures and demo odds are never eligible for a paper bet.
  • Missing, stale, or unsupported data should produce a skip or a blocked readiness state.
  • A real fixture needs a valid active model artifact before it can be considered.
  • Do not use Reset Demo if you want to keep research history; it is a destructive reset.

Actions and troubleshooting

Using the sidebar safely

Fetch Live Odds
Pulls the configured bookmaker feed. Check freshness and errors afterwards.
Pull All Live Data
Runs the broader live-data refresh. It can take longer than a simple odds fetch.
Research commands
Train or rebuild research artifacts. These are not daily actions and may take time.
Action status
Every click reports success, progress, or an error near the top of the dashboard.
When something fails
Read the action message, then open Data Health and Event Log for the provider or workflow detail.
